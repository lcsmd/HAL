FEATURES
========

Text Mode (hal_text_client.py):
- ✅ Simple text-based interface
- ✅ No audio dependencies required
- ✅ Interactive and single-query modes
- ✅ Intent detection (MEDICATION, APPOINTMENT, HEALTH_DATA)
- ✅ Built-in help and examples
- ✅ Easy to use and troubleshoot

Voice Mode (hal_voice_client.py):
- ✅ Wake word detection ("Hey Jarvis", "Computer")
- ✅ Voice Activity Detection (VAD)
- ✅ Audio feedback sounds
- ✅ Interruption handling
- ✅ 10-second follow-up window
- ✅ Keyboard fallback mode

REQUIREMENTS
============

Hardware:
- MacBook Pro (or any Mac)
- Network connection to Windows HAL server
- Microphone (for voice mode only)
- Speakers (for voice mode only)

Software:
- macOS 10.15 or higher
- Python 3.8 or higher (usually pre-installed)
- Network access to Windows server

Services (on Windows):
- HAL Voice Gateway (PY/voice_gateway.py) on port 8768
- QM Voice Listener (BP/VOICE.LISTENER) on port 8767

QUICK START
===========

1. Get Windows IP:
   ipconfig | findstr IPv4

2. Copy package to Mac

3. Run setup:
   bash setup_mac.sh

4. Configure:
   export HAL_GATEWAY_URL=ws://YOUR_WINDOWS_IP:8768

5. Test:
   bash test_connection.sh

6. Use:
   python3 hal_text_client.py

============

  Text I/O                WebSocket (8768)            TCP (8767)               OpenQM

NOTES
=====

- Text mode requires only 'websockets' Python package
- Voice mode requires additional audio packages (portaudio, sounddevice, etc.)
- Both modes connect to same Voice Gateway
- Gateway handles communication with QM Voice Listener
- QM Listener detects intent and routes to appropriate handlers

For voice features, see README.md section "Voice Mode (Optional)"
