WHAT WAS FIXED (BASED ON OPENQM MANUAL)
═══════════════════════════════════════════════════════════════

The original code had INPUT statement syntax that doesn't work in OpenQM.

❌ WRONG SYNTAX (from other BASIC dialects):
   INPUT "Continue anyway (Y/N)? ", answer
   INPUT "Enter name: ": name

✓ CORRECT OpenQM SYNTAX:
   PRINT "Continue anyway (Y/N)? " :
   INPUT answer
   
   PRINT "Enter name: " :
   INPUT name

KEY POINT: The colon (:) after PRINT suppresses the newline, so the
cursor stays on the same line for input. This is documented in the
OpenQM Reference Manual.

INSTALLATION STEPS
═══════════════════════════════════════════════════════════════

STEP 1: START OPENQM
────────────────────

C:\> qm
:

STEP 2: LOG INTO YOUR ACCOUNT
──────────────────────────────

If you already have TRANS account:
   :LOGTO TRANS

If not, create it first:
   :CREATE.ACCOUNT TRANS C:\OpenQM\TRANS
   :LOGTO TRANS

STEP 3: CREATE BP FILE (if it doesn't exist)
─────────────────────────────────────────────

:CREATE.FILE BP DIRECTORY

Or if using TRANS.BP:
:CREATE.FILE TRANS.BP DIRECTORY

STEP 4: ADD PROGRAMS TO BP
───────────────────────────

METHOD A: Copy files to directory
   1. Copy all 7 program files to C:\OpenQM\TRANS\BP\
   2. In QM: BASIC BP *
   3. In QM: CATALOG BP *

METHOD B: Use ED command (paste each program)
   For each of the 7 programs:
   
   :ED BP programname
   (or :ED TRANS.BP programname)
   
   ----: I
   [Paste the entire program content here]
   ----: FI
   
   Repeat for all 7 programs

STEP 5: COMPILE ALL PROGRAMS
─────────────────────────────

:BASIC BP *
(or :BASIC TRANS.BP *)

Expected output:
Compiling BP ANALYZE.FIELDS
***************
0 error(s)
ANALYZE.FIELDS added to local catalogue

Compiling BP CREATE.FILES
*******
0 error(s)
CREATE.FILES added to local catalogue

... (and so on for all 7)

Compiled 7 program(s) with no errors

STEP 6: CATALOG THE PROGRAMS
─────────────────────────────

:CATALOG BP *
(or :CATALOG TRANS.BP *)

STEP 7: CREATE DATABASE FILES
──────────────────────────────

:RUN BP CREATE.FILES
(or :RUN TRANS.BP CREATE.FILES)

Expected output:
Creating system files...
────────────────────────────────────────────────────────
Creating TRANSACTIONS...
File created: TRANSACTIONS (DYNAMIC)
Creating dictionary items...
  - ID
  - TRANS.DATE
  - ORIGINAL.PAYEE
  ... (23 fields total)

Creating PAYEES...
... (etc for all 5 files)

Created 5 files successfully!

STEP 8: IMPORT SAMPLE RULES
────────────────────────────

:RUN BP MANAGE.RULES
(or :RUN TRANS.BP MANAGE.RULES)

Select option: 8 (Import sample rules)
Select option: 9 (Exit)

STEP 9: IMPORT YOUR DATA
────────────────────────

:RUN BP IMPORT.QUICKEN.ENHANCED C:\path\to\your\quicken.csv
(or :RUN TRANS.BP IMPORT.QUICKEN.ENHANCED C:\path\to\your\quicken.csv)

STEP 10: PROCESS AND REPORT
────────────────────────────

:RUN BP STANDARDIZE.PAYEES
:RUN BP TAG.REIMBURSABLE
:RUN BP REPORT.REIMBURSABLE SUMMARY

TROUBLESHOOTING

PROBLEM: Still getting compilation errors
SOLUTION: Make sure you're using the files from THIS package,
          not the original ones. Delete old versions from BP

SOLUTION: Run CREATE.FILES first to create the database

SOLUTION: Use CREATE.ACCOUNT TRANS C:\OpenQM\TRANS first

SOLUTION: Use full paths with forward slashes:
          C:/Users/YourName/Documents/file.csv
