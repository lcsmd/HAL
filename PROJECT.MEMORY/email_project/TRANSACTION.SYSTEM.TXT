INSTALLATION - 3 STEPS:

Step 1: EXTRACT THE ZIP
────────────────────────
unzip openqm_transaction_system.zip
cd /your/qm/directory/

Step 2: CATALOG THE 7 PROGRAMS
────────────────────────────────
qm

At the : prompt, enter each command:

CATALOG CREATE.FILES
CATALOG IMPORT.QUICKEN.ENHANCED
CATALOG STANDARDIZE.PAYEES
CATALOG TAG.REIMBURSABLE
CATALOG REPORT.REIMBURSABLE
CATALOG MANAGE.RULES
CATALOG ANALYZE.FIELDS

Step 3: CREATE DATABASE FILES
──────────────────────────────
RUN CREATE.FILES

You should see:
  Creating TRANSACTIONS - Main transaction data
  [OK] File created successfully
  Creating PAYEES - Standardized payee master
  [OK] File created successfully
  ...etc

If you see this with NO ERRORS → Installation complete!

═══════════════════════════════════════════════════════════════

VERIFY INSTALLATION:

RUN MANAGE.RULES

Should display a menu:
  Rule Management Menu
  ============================================================
  1. List all rules
  2. Add payee standardization rule
  ...

Type 9 to exit.

If the menu appears → Everything is working!

FIRST USE:

1. Import Sample Rules:
   RUN MANAGE.RULES
   Type: 8 (Import sample rules)
   Type: 9 (Exit)

2. Import Your Quicken CSV File:
   RUN IMPORT.QUICKEN.ENHANCED your_transactions.csv

3. Analyze What Was Imported:
   RUN ANALYZE.FIELDS

4. Process the Transactions:
   RUN STANDARDIZE.PAYEES
   RUN TAG.REIMBURSABLE

5. Generate Reports:
   RUN REPORT.REIMBURSABLE SUMMARY
   RUN REPORT.REIMBURSABLE EXPORT

TROUBLESHOOTING:

Q: Compilation errors when cataloging
A: Make sure you're only cataloging the 7 programs listed above.
   Don't try to compile .md, .txt, or .sh files!

Q: "Cannot open TRANSACTIONS file"
A: Run CREATE.FILES first to create the database files.

Q: "Program not found"
A: Make sure you CATALOG the program before trying to RUN it.

Q: Need to see what fields were imported
A: Run ANALYZE.FIELDS to see field usage statistics.
